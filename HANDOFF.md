# 조디악 럼블 — 작업 핸드오프 (Handoff)

> 이 문서는 **새 세션에서 `party-rift` 리포 작업을 이어가기 위한 컨텍스트 프라이머**입니다.
> 새 세션에 이 파일을 주고 "이 내용을 기준으로 작업해줘"라고 요청하면 됩니다.

---

## 1. 이 프로젝트가 뭔가

**조디악 럼블 (ZODIAC RUMBLE)** — 구(舊) 파티 리프트, 리포 폴더 이름은 `party-rift` 유지 —
3D AOS(MOBA) 파티 게임. 친구들이 각자 자기 폰/태블릿으로
**온라인 방**에 모여 **3:3 / 5:5** 전장에서 타워를 부수고 **넥서스**를 터뜨리면 승리.
7인치 휴대기기 **가로 와이드** 기준. 기기마다 조이스틱이 필요해 **온라인 방 전용**.

- 스택: **React + Vite** (프론트) / **Node + ws** (온라인 서버) / **three.js** (3D, 전장 진입 시 동적 로딩)
- 동기화: **서버 권위(server-authoritative)** — 서버가 큐→드래프트→전투 전 생애주기를 주도.
  전투는 60Hz 시뮬 + 30Hz 바이너리 델타 스냅샷 방송, 클라는 내 영웅 입력 예측 + 남 유닛 보간
- 접속: **매치메이킹 큐**(방 코드 없음). 모드(3:3/5:5) 선택 → 입장 → 목표 인원 차거나 1분이면 시작(빈자리 봇).
  랜덤 팀 + **스네이크 드래프트**(직업 20종 픽). deviceId로 새로고침/끊김 시 진행 복구.
- 캐릭터: 직업 **20종** 드래프트 픽, 아바타는 **12지신 이모지** 자동 배정(추후 이미지 교체 가능)
- 사운드: **Web Audio API** 생성(에셋 파일 없음)

## 2. 이 리포의 출생 배경 (중요)

이 리포는 원래 **여러 게임 모음(BoardGameParty)** 에서 **리프트 게임만 추출**해 만든 단독 리포다.
즉 git 히스토리에는 제거된 8개 게임(사다리/메모리/도블/스릴팡/달리기/두더지/신호등/카트)의
과거 커밋이 남아 있지만, **현재 코드에는 리프트만 존재**한다.

### 단독화 작업으로 한 일 (한 커밋: "파티 리프트를 단독 게임으로 분리")
- 다른 게임 8종 폴더 + 로컬 핫시트 로비(`GameLobby.jsx`, `lobby/games.js`) + 로컬 roster 저장 제거
- `App.jsx`: 흐름을 **홈(방 만들기/코드 참여) → 온라인 로비 → 전투** 한 갈래로 축소
  (리프트는 온라인 전용이라 로컬/핫시트 모드 자체를 삭제)
- `HomeScreen` / `OnlineLobby`: 리프트 브랜딩, 게임 선택 그리드 → 단일 **"⚔️ 전투 시작!"** 버튼
- 방 정원 5 → **10명** (5:5 모드까지 사람으로 채울 수 있게; `server/rooms.js`, `OnlineLobby.jsx`)
- 서버 게임 등록소(`server/games.js`)·테스트를 리프트만으로 정리(카트 의존 테스트 제거)
- 메타데이터 리브랜딩(`package.json` name → `party-rift`, `index.html` title, 서버 로그)
- README 단독 게임용 재작성, 옛 컬렉션 설계 노트(KEYPOINT.md/PLAN.md) 제거

**검증 상태**: 유닛 테스트 **119개 전부 통과**, 프로덕션 빌드 정상(리프트 3D는 lazy 청크 분리).

## 3. 실행 / 테스트 / 빌드

```bash
npm install
npm run dev      # 개발: 룸 서버(8787) + vite(5173) 동시 실행
npm run dev:web  # 프론트만 (온라인 서버 제외)
npm run build    # 프로덕션 빌드 → dist/
npm start        # 빌드된 앱 + 룸 서버를 한 포트(8787)로 서빙
npm test         # 게임/룸 로직 유닛 테스트 (node --test, 119개)
```

브라우저를 **가로 방향**으로(세로면 회전 안내). 같은 와이파이의 다른 기기는 콘솔의
`Network: http://<내IP>:5173/` 로 접속, `/ws`는 vite가 룸 서버로 프록시하므로 주소 하나만 공유하면 됨.

## 4. 폴더 구조 / 핵심 파일

```
server/
├─ index.js         # HTTP 정적 서빙 + /ws (큐→드래프트→플레이 생애주기 스케줄러)
├─ matchmaking.js   # 모드별 매치메이킹 큐 순수 로직 — 테스트: matchmaking.test.js
├─ match.js         # 팀 배정 / 스네이크 드래프트 / 픽 순수 로직 — 테스트: match.test.js
├─ realtime.js      # 서버 권위 실시간 세션(60Hz 시뮬 / 30Hz 방송) — 테스트: realtime.test.js
└─ games.js         # 실시간 게임 어댑터 등록소 ({ rift })
src/
├─ App.jsx          # 화면 흐름(대문 → 큐 → 드래프트 → 전투), 서버 상태로 라우팅
├─ main.jsx
├─ net/             # 온라인 동기화
│  ├─ RoomClient.js / MatchContext.jsx   # ws 클라이언트(얇은 래퍼) + 매치 상태 컨텍스트
│  ├─ useRealtimeGame.js                 # 실시간 게임 입력/스냅샷 훅
│  └─ realtime/  (sim.js, codec.js + .test.js)  # 고정 STEP 시뮬, 바이너리 델타 코덱
├─ lobby/
│  ├─ SshGate.jsx       # 접속 대문(SSH 스타일) + 모드 선택 + 대기열 입장
│  ├─ QueueScreen.jsx   # 매칭 대기(인원/남은 시간)
│  └─ DraftScreen.jsx   # 스네이크 드래프트(팀 보드 + 직업 픽 + 10초 타이머)
├─ shared/          # zodiac.js, sound.js, storage.js(리프트 조작 설정), Fireworks, FullscreenButton, ErrorBoundary 등
└─ games/rift/      # ⭐ 게임 본체
   ├─ engine.js        # 순수 게임 로직(영웅/스킬/미니언/타워/넥서스/골드/아이템) — 테스트: engine.test.js
   ├─ engine.test.js
   ├─ netgame.js       # 서버 권위 어댑터(createGame/setInput/step/makeView/STEP)
   ├─ RiftGame.jsx     # 게임 메인(상태/설정/UI 조립)
   └─ (3D 씬 / HUD / 조이스틱 / 상점 등 컴포넌트)
```

설계 원칙: **순수 로직(engine.js) + 유닛 테스트**를 화면과 분리. 룸 로직도 `server/rooms.js`로 분리/테스트.

## 5. 게임 규칙 요약 (engine 기준)

- **3:3 / 5:5 모드** — 빈자리는 봇이 채워 항상 동일 인원. 5:5는 더 큰 맵 + 정글 캠프 증가.
- **3갈래 레인**(외곽→내곽 타워→넥서스), 미니언 웨이브 + 🐺정글몹, 팀 버프 주는 🐉용/👹바론.
- **직업 14종**: 기본 6종(⚔️전사 🏹궁수 🔮마법사 💚힐러 🥷암살자 🛡️탱커) + 신규 8종(❄️한빙술사 🪓검투사 ☠️주술사 🔰수호기사 🗡️검성 🪝사슬잡이 🐺야수조련사 🔧엔지니어). 한 팀에 같은 직업 1명만. 마법사는 빙결을 한빙술사에 넘기고 순수 폭발 누커로 리워크됨. 메커니즘: 빙결(한빙)·흡혈(검투사)·DoT/안티힐(주술사)·아군보호막(수호기사)·패링(검성)·끌어오기/속박(사슬잡이)·소환펫(야수조련사)·설치포탑(엔지니어).
- **🌿수풀 은신 + 전장의 안개**(시야 밖 적은 미니맵에도 안 보임).
- 경험치로 **최대 Lv18**(보조 스킬 Lv3·궁극기 Lv5 해금).
- **💰골드 & 상점** — 우물에서 🛒상점, **아이템 24종**을 인벤토리 3칸에 구매.
- 20분 안에 못 끝내면 타워/킬 점수 판정.

## 6. 알려진 제한 (온라인 v2 — 매치메이킹)

- 상태는 **서버 메모리**에만 있음 → 서버 재시작 시 진행 중 큐/매치 소실(같은 프로세스 내 deviceId 재접속만 복구).
- 전투 중 끊긴 사람은 약 **20초 유예**(PLAY_GRACE_MS) 후 봇이 인계(그 안에 재접속하면 복귀).
- 드래프트/카운트다운 중 끊겨도 자리는 유지되고, 본인 차례면 10초 후 자동 픽(재접속 시 복귀).
- 호스트/방 코드/일시정지/한판더(리매치) 개념은 제거됨(서버 주도 단일 매치). 종료 후 "새 매치 찾기"로 대문 복귀.
- 빌드 시 RiftGame 청크가 500kB 경고 — three.js 포함 탓(이미 lazy 분리됨, 기능 문제 아님).

## 7. 새 세션에서 이어가면 좋을 후보 작업 (아이디어, 미확정)

- 게임 중 재접속/방 유지 견고화(새로고침해도 자리 보존).
- 5:5 정글/오브젝트 밸런스 튜닝, 직업/아이템 수치 밸런싱.
- 캐릭터 이모지 → 커스텀 이미지(PNG/SVG) 교체 포인트 정리.
- 번들 추가 코드 스플릿(three.js) / 로딩 UX 개선.
- 관전 모드, 리매치 시 팀/직업 유지 옵션.
- CI(테스트+빌드) 워크플로 추가.

> 위 항목은 확정된 요구사항이 아니라 후보다. 실제 작업은 사용자 요청을 우선한다.
