# ⚡ 조디악 러쉬 (ZODIAC RUSH)

> 구(舊) 파티 리프트. 리포 폴더/원격 이름은 `party-rift`를 유지한다.

3D **AOS(MOBA)** 파티 게임. 친구들과 각자 자기 폰/태블릿으로 **온라인 방**에 모여
**3:3 / 5:5** 전장에서 타워를 부수고 **넥서스**를 터뜨리면 승리!
7인치 휴대기기 **가로 와이드** 화면 기준으로 만들었고, 기기마다 조이스틱이 필요해 **온라인 방 전용**입니다.

> 캐릭터는 **12지신 이모지**로 표현합니다. 한 기기당 1명만 영웅을 조종하고, 빈자리는 봇이 채웁니다.

## 게임 소개

- **3:3 / 5:5 모드 선택** — 빈자리는 봇이 채워 항상 같은 인원. 5:5는 더 큰 맵 + 정글 캠프 증가.
- **3갈래 레인**(외곽→내곽 타워→**넥서스**가 터지면 승리), 미니언 웨이브와 🐺 정글몹,
  잡으면 팀 버프를 주는 🐉 용 / 👹 바론.
- **직업 20종** — ⚔️전사 · 🏹궁수 · 🔮마법사 · 💚힐러 · 🥷암살자 · 🛡️탱커 · ❄️한빙술사 · 🪓검투사 · ☠️주술사 · 🔰수호기사 · 🗡️검성 · 🪝사슬잡이 · 🐺야수조련사(소환) · 🔧엔지니어(포탑) · 🌿넝쿨사냥꾼(속박) · 🌬️돌풍술사(넉백) · ⏳시간술사(되감기) · 👻공포술사(공포) · 🎭환영무희(분신) · 🪨대지술사(돌벽). 각 직업은 시그니처가 겹치지 않게 설계.
  스킬 피해는 직업 주력 스탯에 비례하고, 한 팀에 같은 직업은 한 명만.
- **🌿 수풀 은신 + 전장의 안개** — 시야 밖 적은 미니맵에도 안 보입니다.
- 경험치로 **최대 Lv18** 성장(보조 스킬 Lv3·궁극기 Lv5 해금).
- **💰 골드 & 상점** — 미니언·정글·타워·킬로 골드를 모아 우물에서 🛒 상점을 열어
  **아이템 24종**을 인벤토리 3칸에 구매.
- 20분 안에 못 끝내면 타워/킬 점수로 판정.

## 기술 스택

- **React + Vite** (프론트), **Node + ws** (온라인 룸 서버)
- 온라인 동기화: **서버 권위(server-authoritative)** 모델 — 서버가 큐→드래프트→전투의 전 생애주기를 주도
  - **매치메이킹 큐**(방 코드 없음): 모드(3:3/5:5)를 골라 입장, 목표 인원이 차거나 1분이 지나면 시작(빈자리는 봇)
  - **랜덤 팀 + 스네이크 드래프트**: 블루1·레드1·레드2·블루2… 순으로 직업을 픽(사람 10초/봇 자동, 같은 팀 직업 중복 불가)
  - 전장은 서버가 60Hz로 시뮬레이션하고 30Hz로 **바이너리 델타 스냅샷**을 방송,
    클라이언트는 내 영웅을 입력 즉시 예측·보정하고 남의 유닛은 보간으로 부드럽게 표현
  - **재접속 복구**: 같은 기기(deviceId)면 새로고침/끊김에도 진행 중이던 큐/드래프트/전투로 이어진다
- 직업 **20종**을 드래프트로 픽(빙결/흡혈/DoT/보호막/패링/끌어오기/소환/포탑/넉백/되감기/공포/분신/지형 등 고유 메커니즘) · 아바타는 **12지신 이모지** 자동 배정
- 사운드: **Web Audio API**로 생성 (에셋 파일 불필요)
- 3D는 **three.js** — 전장에 들어갈 때만 동적 로딩(번들 분리)
- 게임 로직은 **순수 함수 + 유닛 테스트**로 화면과 분리 (`engine.js`, `server/matchmaking.js`, `server/match.js`)

## 실행

```bash
npm install
npm run dev      # 개발: 룸 서버(8787) + vite(5173)를 한 번에 실행
npm run dev:web  # 프론트만 (온라인 서버 제외)
npm run build    # 프로덕션 빌드 → dist/
npm start        # 빌드된 앱 + 룸 서버를 한 포트(8787)로 서빙
npm test         # 게임/룸 로직 유닛 테스트(node --test)
```

브라우저를 **가로 방향**으로 두고 즐기세요(세로면 회전 안내 표시). 우상단 **⛶ 전체화면** 버튼 권장.

### 같은 네트워크/외부 기기에서 접속

`npm run dev` 실행 시 콘솔의 `Network: http://<내IP>:5173/` 로 같은 와이파이의 폰/태블릿에서 접속할 수 있습니다.
온라인 방 WebSocket(`/ws`)은 vite가 룸 서버로 프록시하므로 **주소 하나(5173)** 만 공유하면 됩니다.
배포 시에는 `npm run build && npm start` 로 한 포트(8787)에서 정적 파일과 WebSocket을 함께 서빙합니다.

### 알려진 제한 (온라인 v2)

- 상태는 서버 메모리에만 있습니다. **서버를 재시작하면** 진행 중이던 큐/매치는 사라집니다(기기 재접속 복구는 같은 서버 프로세스 내에서만).
- 전투 중 끊긴 사람은 약 20초 유예 후 봇이 인계합니다(그 안에 재접속하면 복귀).

## 플레이 흐름

1. **대문(SSH 스타일)** 에서 모드(3:3 / 5:5)를 고르고 **대기열 입장**.
2. **큐**: 같은 모드끼리 매칭. 목표 인원이 차면 즉시, 첫 입장 후 **1분**이 지나면 빈자리를 봇으로 채워 시작.
3. **드래프트**: 랜덤 팀 배정 → 스네이크 순서(블루1·레드1·레드2·블루2·블루3·레드3…)로 직업 픽. 사람 10초 제한, 봇 자동, 같은 팀 직업 중복 불가.
4. 전원 픽 완료 → 전장 진입, 엔진의 **“이제 곧 경기가 시작합니다” 3초 카운트다운** 후 본게임.
5. 넥서스가 터지면 폭죽 연출 → **새 매치 찾기**(대문으로).

## 폴더 구조

```
server/
├─ index.js         # HTTP 정적 서빙 + /ws (큐→드래프트→플레이 생애주기 주도)
├─ matchmaking.js   # 모드별 매치메이킹 큐 순수 로직 — 테스트: matchmaking.test.js
├─ match.js         # 팀 배정 / 스네이크 드래프트 / 픽 순수 로직 — 테스트: match.test.js
├─ realtime.js      # 서버 권위 실시간 세션(60Hz 시뮬 / 30Hz 방송)
└─ games.js         # 실시간 게임 어댑터 등록소(rift)
src/
├─ App.jsx          # 화면 흐름(대문 → 큐 → 드래프트 → 전투), 서버 상태로 라우팅
├─ net/             # 온라인 동기화(RoomClient/MatchContext/useRealtimeGame, realtime 코덱/sim)
├─ lobby/           # SshGate(대문) · QueueScreen(큐) · DraftScreen(드래프트)
├─ shared/          # 공통 모듈 (zodiac, sound, storage, Fireworks, …)
└─ games/rift/      # 리프트 (engine.js + engine.test.js + 3D/UI 컴포넌트 + netgame 어댑터)
```

순수 로직(`engine.js`, `server/matchmaking.js`, `server/match.js`)과 테스트는 화면과 분리돼 있습니다.
