# ⚔️ 파티 리프트 (Party Rift)

3D **AOS(MOBA)** 파티 게임. 친구들과 각자 자기 폰/태블릿으로 **온라인 방**에 모여
**3:3 / 5:5** 전장에서 타워를 부수고 **넥서스**를 터뜨리면 승리!
7인치 휴대기기 **가로 와이드** 화면 기준으로 만들었고, 기기마다 조이스틱이 필요해 **온라인 방 전용**입니다.

> 캐릭터는 **12지신 이모지**로 표현합니다. 한 기기당 1명만 영웅을 조종하고, 빈자리는 봇이 채웁니다.

## 게임 소개

- **3:3 / 5:5 모드 선택** — 빈자리는 봇이 채워 항상 같은 인원. 5:5는 더 큰 맵 + 정글 캠프 증가.
- **3갈래 레인**(외곽→내곽 타워→**넥서스**가 터지면 승리), 미니언 웨이브와 🐺 정글몹,
  잡으면 팀 버프를 주는 🐉 용 / 👹 바론.
- **직업 6종** — ⚔️전사 · 🏹궁수 · 🔮마법사 · 💚힐러 · 🥷암살자 · 🛡️탱커.
  스킬 피해는 직업 주력 스탯에 비례하고, 한 팀에 같은 직업은 한 명만.
- **🌿 수풀 은신 + 전장의 안개** — 시야 밖 적은 미니맵에도 안 보입니다.
- 경험치로 **최대 Lv18** 성장(보조 스킬 Lv3·궁극기 Lv5 해금).
- **💰 골드 & 상점** — 미니언·정글·타워·킬로 골드를 모아 우물에서 🛒 상점을 열어
  **아이템 24종**을 인벤토리 3칸에 구매.
- 20분 안에 못 끝내면 타워/킬 점수로 판정.

## 기술 스택

- **React + Vite** (프론트), **Node + ws** (온라인 룸 서버)
- 온라인 동기화: **서버 권위(server-authoritative)** 모델
  - 로비(방 코드/참가자/화면 전환)는 서버(`server/`)가 관리
  - 전장은 서버가 60Hz로 시뮬레이션하고 20Hz로 **바이너리 델타 스냅샷**을 방송,
    클라이언트는 내 영웅을 입력 즉시 예측·보정하고 남의 유닛은 보간으로 부드럽게 표현
- 캐릭터: **12지신 이모지** (추후 이미지 교체 가능)
- 사운드: **Web Audio API**로 생성 (에셋 파일 불필요)
- 3D는 **three.js** — 전장에 들어갈 때만 동적 로딩(번들 분리)
- 게임 로직은 **순수 함수(engine.js) + 유닛 테스트**로 화면과 분리 (룸 로직도 `server/rooms.js`로 분리/테스트)

## 실행

```bash
npm install
npm run dev      # 개발: 룸 서버(8787) + vite(5173)를 한 번에 실행
npm run dev:web  # 프론트만 (온라인 서버 제외)
npm run build    # 프로덕션 빌드 → dist/
npm start        # 빌드된 앱 + 룸 서버를 한 포트(8787)로 서빙
npm test         # 게임/룸 로직 유닛 테스트(node --test)
```

브라우저를 **가로 방향**으로 두고 즐기세요(세로면 회전 안내 표시). 우상단 **⛶ 전체화면** 버튼 권장.

### 같은 네트워크/외부 기기에서 접속

`npm run dev` 실행 시 콘솔의 `Network: http://<내IP>:5173/` 로 같은 와이파이의 폰/태블릿에서 접속할 수 있습니다.
온라인 방 WebSocket(`/ws`)은 vite가 룸 서버로 프록시하므로 **주소 하나(5173)** 만 공유하면 됩니다.
배포 시에는 `npm run build && npm start` 로 한 포트(8787)에서 정적 파일과 WebSocket을 함께 서빙합니다.

### 알려진 제한 (온라인 v1)

- 게임 도중 기기를 새로고침하면 그 기기 참가자는 방에서 빠집니다(로비에서 다시 등록).
- 호스트 기기가 나가거나 연결이 끊기면 방이 닫힙니다.

## 플레이 흐름

1. 홈에서 **방 만들기**(이 기기가 호스트) 또는 **코드로 참여**.
2. 온라인 로비에서 각자 자기 기기에서 참가자 추가(이름 + 12지신 말, 중복 불가).
   한 명이 방을 만들면 **4글자 코드**가 생기고, 친구들은 같은 주소로 접속해 코드를 입력
   (또는 초대 링크 `?room=CODE`)하면 합류합니다.
3. 호스트가 **⚔️ 전투 시작!** → 팀/직업 설정 → 플레이. 빈자리는 봇이 채웁니다.
4. 넥서스가 터지면 폭죽 연출 → 한판 더(호스트) / 로비로.

## 폴더 구조

```
server/
├─ index.js        # HTTP 정적 서빙 + /ws 룸 서버(릴레이)
├─ rooms.js        # 방/참가자 순수 로직
├─ realtime.js     # 서버 권위 실시간 세션(60Hz 시뮬 / 20Hz 방송)
└─ games.js        # 실시간 게임 어댑터 등록소(rift)
src/
├─ App.jsx         # 화면 흐름(홈 → 온라인 방 → 전투)
├─ net/            # 온라인 동기화(RoomClient/RoomContext/useRealtimeGame, realtime 코덱/sim)
├─ lobby/          # HomeScreen(모드 선택) · OnlineLobby(방 로비)
├─ shared/         # 공통 모듈 (zodiac, sound, storage, Fireworks, …)
└─ games/rift/     # 리프트 (engine.js + engine.test.js + 3D/UI 컴포넌트 + netgame 어댑터)
```

`engine.js`(순수 로직)와 `engine.test.js`(테스트)는 화면과 분리되어 있고, 룸 로직도 `server/rooms.js`로 분리/테스트됩니다.
