# 🎉 보드게임 파티 (BoardGameParty)

여러 보드/파티게임을 한 곳에 모아 즐기는 컬렉션.
**한 태블릿(핫시트)** 으로도, **각자 자기 폰/태블릿으로 온라인 방에 모여서도** 플레이할 수 있습니다.
7인치 휴대기기 **가로 와이드** 화면 기준으로 만들었고, 어린이(6세+)도 쉽게 즐길 수 있게 직관적인 조작과 멀티터치를 지원합니다.

> 캐릭터는 **12지신 이모지**로 표현합니다. 참가자는 로비에서 한 번 정해두면 모든 게임이 공유해요.

**모든 게임이 두 모드를 지원합니다.** (단, 🏎️ 파티 카트와 ⚔️ 파티 리프트는 기기마다 조이스틱이 필요해 **온라인 방 전용** — 기기당 1명만 조종)

- 📱 **한 기기로 놀기**: 기존 핫시트 + 멀티터치. 참가자는 localStorage에 저장.
- 🌐 **온라인 방**: 한 명이 방을 만들면 **4글자 코드**가 생깁니다. 친구들은 같은 주소로 접속해
  코드를 입력(또는 초대 링크 `?room=CODE`)하면 합류. **한 기기에서 여러 명** 등록도 되므로
  "거실 태블릿 2명 + 친구 폰 2명" 같은 혼합 구성도 가능합니다.

## 기술 스택

- **React + Vite** (프론트), **Node + ws** (온라인 룸 서버)
- 온라인 동기화: **호스트 권위(host-authoritative)** 모델
  - 로비(방 코드/참가자/화면 전환)는 서버(`server/`)가 관리
  - 게임 진행은 호스트 기기가 계산해 `state`로 전파, 게스트 입력은 `action`으로 호스트에 릴레이
  - 반응속도류 게임(신호등/스릴팡)은 **각자 기기 화면 기준으로 시간을 측정**해 네트워크 지연에도 공평
- 한 기기에서 **핫시트 + 멀티터치**(여러 명이 동시에 자기 영역 터치) — `onPointerDown` 사용
- 캐릭터: **12지신 이모지** (추후 이미지 교체 가능)
- 사운드: **Web Audio API**로 생성 (에셋 파일 불필요)
- 3D 게임(파티 카트/파티 리프트)은 **three.js** — 게임을 고를 때만 동적 로딩(번들 분리)
- 게임 로직은 **순수 함수(engine.js) + 유닛 테스트**로 화면과 분리 (룸 로직도 `server/rooms.js`로 분리/테스트)

## 실행

```bash
npm install
npm run dev      # 개발: 룸 서버(8787) + vite(5173)를 한 번에 실행
npm run dev:web  # 프론트만 (온라인 모드 제외)
npm run build    # 프로덕션 빌드 → dist/
npm start        # 빌드된 앱 + 룸 서버를 한 포트(8787)로 서빙
npm test         # 게임/룸 로직 유닛 테스트(node --test)
```

브라우저를 **가로 방향**으로 두고 즐기세요(세로면 회전 안내 표시). 우상단 **⛶ 전체화면** 버튼 권장.

### 같은 네트워크/외부 기기에서 접속

`npm run dev` 실행 시 콘솔의 `Network: http://<내IP>:5173/` 로 같은 와이파이의 폰/태블릿에서 접속할 수 있습니다.
온라인 방 WebSocket(`/ws`)은 vite가 룸 서버로 프록시하므로 **주소 하나(5173)** 만 공유하면 됩니다.
배포 시에는 `npm run build && npm start` 로 한 포트(8787)에서 정적 파일과 WebSocket을 함께 서빙합니다.

### 알려진 제한 (온라인 v1)

- 게임 도중 기기를 새로고침하면 그 기기 참가자는 방에서 빠집니다(로비에서 다시 등록).
- 호스트 기기가 나가거나 연결이 끊기면 방이 닫힙니다.
- 온라인 모드에서는 차례 정하기(주사위) 화면 대신 **무작위 순서**로 시작합니다.

## 플레이 흐름

1. 홈에서 모드 선택: **한 기기로 놀기** / **온라인 방 만들기** / **코드로 참여**.
2. 로비에서 참가자 추가(이름 + 12지신 말, 중복 불가) — 온라인이면 각자 자기 기기에서 추가. 인원 미달 게임은 비활성.
3. 게임 선택(온라인은 호스트가) → 게임별 설정(보드 크기/난이도 등) → 플레이.
4. 우승 시 폭죽 연출 → 다시하기(호스트) / 로비로.

## 폴더 구조

```
server/
├─ index.js                # HTTP 정적 서빙 + /ws 룸 서버(릴레이)
├─ rooms.js                # 방/참가자 순수 로직
└─ rooms.test.js
src/
├─ App.jsx                 # 화면 흐름(홈 → 한기기/온라인 → 게임)
├─ net/                    # 온라인 동기화
│  ├─ RoomClient.js        # WebSocket 클라이언트 래퍼 + 기기 ID
│  ├─ RoomContext.jsx      # 방 상태 컨텍스트(RoomProvider/useRoom)
│  ├─ useGameNet.js        # 게임용 훅: 호스트 publish / 게스트 action
│  └─ NetParts.jsx         # 대기 화면 등 공용 조각
├─ lobby/
│  ├─ HomeScreen.jsx       # 모드 선택(한 기기/방 만들기/코드 참여)
│  ├─ GameLobby.jsx        # 한 기기 모드 로비
│  ├─ OnlineLobby.jsx      # 온라인 방 로비(코드/기기별 참가자)
│  └─ games.js             # 게임 목록(두 로비 공유)
├─ shared/                 # 공통 모듈 (zodiac, Dice, sound, storage, Fireworks, …)
└─ games/                  # 게임별 (engine.js + *.test.js + UI)
   ├─ ladder/  memory/  dobble/  thrillpang/  race/  whack/  traffic/
```

각 게임 컴포넌트는 **호스트(로직) / 게스트(미러링) / 공용 표현 컴포넌트** 구조로,
`engine.js`(순수 로직)와 `engine.test.js`(테스트)는 그대로 화면과 분리되어 있습니다.
