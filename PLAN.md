# BoardGameParty — 작업 계획 (Work Plan)

> 기준 결정: **React + Vite**, **핫시트(한 기기)**, **이모지 캐릭터**, **가로 와이드 7인치**.
> 첫 게임: **사다리 게임**.

---

## 0. 목표 폴더 구조 (확장 고려)

```
BoardGameParty/
├─ index.html
├─ package.json
├─ vite.config.js
├─ public/
│  └─ (사운드/이미지 에셋 - 추후)
├─ src/
│  ├─ main.jsx
│  ├─ App.jsx                 # 라우팅: 로비 ↔ 게임
│  ├─ styles/                 # 공통 스타일, 가로 와이드 레이아웃
│  ├─ lobby/
│  │  └─ GameLobby.jsx        # 게임 목록(컬렉션 진입점)
│  ├─ shared/                 # 게임 공통 컴포넌트(주사위, 캐릭터, 버튼 등)
│  │  ├─ Dice.jsx
│  │  └─ zodiac.js            # 12지신 데이터(이모지)
│  └─ games/
│     └─ ladder/              # 사다리 게임
│        ├─ LadderGame.jsx    # 게임 메인/상태관리
│        ├─ Board.jsx         # 뱀형 보드 렌더링
│        ├─ Pawn.jsx          # 말
│        ├─ board.config.js   # 칸 수/발판 위치/하강폭 등 설정
│        └─ engine.js         # 순수 게임 로직(이동/발판/승리 판정)
```

핵심: **게임 로직(engine)과 화면(컴포넌트) 분리** → 테스트 쉽고, 게임 추가 시 패턴 재사용.

---

## 1. 마일스톤

### M1. 프로젝트 셋업
- Vite + React 초기화, 의존성 설치.
- 가로 와이드(16:9~16:10) 기본 레이아웃 + 세로 안내.
- 로비(게임 목록) 골격 → "사다리 게임" 카드 1개.

### M2. 사다리 게임 — 핵심 로직 (engine.js)
- 보드 설정(칸 수, 발판 맵) 데이터화.
- 이동/발판 적용/하강폭 제한/승리 판정 순수 함수.
- 간단한 유닛 테스트(선택).

### M3. 게임 셋업 화면
- 인원 수 선택(2~6).
- 12지신 말 선택(중복 불가), 선택 순서 = 턴 순서.

### M4. 보드 렌더링
- **뱀(S자) 형태** 칸 배치 + 칸 번호 + 진행 방향 표시.
- 올라가는/내려가는 발판 시각화.
- 7인치 가로 화면에 꽉 차게.

### M5. 주사위 & 턴 진행
- 주사위 **굴러가는 연출** → 멈추면 **중앙 큰 숫자 애니메이션**.
- 말 **한 칸씩 이동** 애니메이션.
- 발판 발동 연출(↑/↓ 강조).
- 현재 턴 표시 / 턴 넘김.

### M6. 승리 & 마무리
- 골인 판정(초과 허용 기준) + 우승 연출.
- 다시하기 / 로비로.

### M7. 다듬기 (후순위)
- 사운드 효과 + on/off.
- 반응형/가독성 튜닝, 접근성(큰 버튼).
- 에셋(이모지→이미지) 교체 포인트 정리.

---

## 2. 진행 순서 제안

1. **M1 셋업**을 먼저 만들어 화면이 뜨는 상태 확보.
2. **M2 로직**으로 규칙 확정값 코드화 (KEYPOINT의 "확정 필요" 답변 반영).
3. M3~M6 순서로 플레이 가능한 1차 버전 완성.
4. M7은 1차 플레이 확인 후 진행.

> 각 마일스톤 완료 시 커밋하고, 1차 플레이 가능 시점에 한 번 점검받기.
