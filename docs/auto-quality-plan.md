# 저사양 자동 그래픽 품질 — 구현 계획

작성: 2026-07-11. 출시 전 체크리스트 5번(저사양 대응) 설계 문서.
현재 상태: 그래픽 품질은 수동 3단(상/중/하, `bgp.rift.gfx.v1`)이고 기본값은 "중".

## 목표

첫 실행하는 저가 폰에서 "버벅이는 첫인상"을 없앤다. 유저가 설정을 몰라도
기기에 맞는 품질로 시작하고, 수동 선택은 언제나 자동을 이긴다.

## 설계 (2단계)

### 1단계 — 첫 매치 실측 기반 자동 결정 (권장, 소규모)

- **측정**: 첫 전투 시작 후 5~15초 구간(카운트다운·로딩 스파이크 제외)의
  프레임 시간을 rAF에서 수집해 **중앙값 FPS**를 구한다 (Rift3D의 렌더 루프에 계측 추가).
- **판정** (기본 품질 "중"에서 시작):
  - 중앙값 < 24fps → "하"로 강등 + 토스트("성능을 위해 그래픽을 낮췄어요 — 설정에서 바꿀 수 있어요")
  - 중앙값 > 55fps → "상"으로 승격 (조용히)
  - 그 외 → "중" 유지
- **저장**: 결과를 `bgp.rift.gfx.auto.v1`에 `{ decided: true, gfx }`로 기록 —
  한 번 정하면 다시 측정하지 않는다(기기당 1회).
- **수동 우선**: 유저가 설정에서 품질을 바꾸면 `manual: true`를 함께 저장하고
  자동 판정은 영원히 개입하지 않는다.
- **매치 중 강등 안전망**: 전투 중 10초 연속 중앙값 < 18fps면 한 단계 강등
  (한 매치에 최대 1회, 토스트 표시). 저가 기기 발열로 인한 후반 급락 대응.

예상 작업량: Rift3D 계측 훅 + storage 키 + RiftGame 결정 로직 + 토스트. 반나절.

### 2단계 — 품질 단계가 실제로 싸게 만드는 것 검증/보강 (별도)

현재 상/중/하가 바꾸는 항목을 실측으로 검증하고, 부족하면 보강:
- 픽셀 비율 캡(하: `devicePixelRatio` 1.0 고정, 중: 1.5, 상: 제한 없음) — WebView에서 효과가 가장 큼
- 그림자/후처리 없음(이미 없음), 파티클 수 반토막(하), 안개 갱신 주기 축소(이미 스로틀)
- 장식 지오메트리(나무·바위 병합군) LOD는 [[rift-graphics-optimization]] 메모 참고 — 게임플레이 요소(영웅/스킬/투사체)는 절대 줄이지 않는 원칙

## 왜 "기기 스펙 감지"가 아니라 실측인가

`navigator.hardwareConcurrency`·GPU 문자열은 WebView에서 신뢰도가 낮고
(같은 SoC라도 발열·절전 상태로 편차 큼), 실측 15초가 가장 정확하다.
메뉴 배경 디오라마(MenuStage)에서 미리 측정하는 방법도 있으나, 실전 부하
(유닛 12+·이펙트)와 달라 과대평가 위험 → 첫 전투 실측이 정답.

## 수용 기준

- 저가 폰(예: 갤럭시 A2x대)에서 첫 매치 후반부 평균 30fps 이상
- 자동 강등 후에도 유저가 설정에서 "상"을 고르면 그대로 유지
- 데스크톱(Electron)에선 사실상 항상 "상" 승격
