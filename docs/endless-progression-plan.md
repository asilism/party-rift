# 무한 방어 심화 — 아이템 강화 + 조디악 증강 카드 설계

작성: 2026-07-22. 상태: **시스템 A(아이템 강화) 1차 구현 완료(PR #224, v49). 시스템 B(증강)는 미착수.**
목적: 30웨이브 이후의 "벽"(레벨 상한 18 + 아이템 5칸)을 넘어 성장이 계속되게 한다.

> **구현 노트(강화, 2026-07-23)**: `h.itemPlus`/`h.itemFails` 상태, `sumStats(items, plus)` 강화 반영,
> `enhanceItem`(성공률/비용 테이블·pity·실패=골드만), 봇 자동강화(`botEnhance`), makeView·net action(`enhance`),
> 상점 UI(강화 버튼·+N 배지·성공/실패 반짝·사운드). 시뮬: 봇 도달 중앙 33→36·꼬리 52웨, 강화 발생 6/10판.
> 시작 계수는 아래 테이블대로 반영됨(PLUS_STAT 0.07, +1 95%/300G … 하드캡 +10) — 실기기 피드백으로 튜닝 예정.

## 배경 / 문제

무한 방어는 난이도(미니언·그림자·보스)가 무한히 오르는데 플레이어 성장은 상한이 있다:
[`engine.js`](../src/games/rift/engine.js) `MAX_LEVEL = 18`에서 XP가 잠기고(`gainXp`가 18에서 xp=0),
아이템은 `ITEM_SLOTS = 5` 조합 종료템이 끝이다. → 설계상 반드시 지는 레이스이고, 30웨 이후가 정체된다.

**해법 방향**: 벽을 "숫자로 미는"(강화) 것 + "새 놀이로 바꾸는"(증강) 것 두 축.
강화는 즉효 골드 배출구, 증강은 리플레이성을 주는 헤드라이너. 리더보드는 이번 스코프에서 제외(추후).

## 공통 설계 원칙

1. **런 한정 상태(run-scoped)**: 강화 수치·증강 카드는 방어전 한 판 안에서만 유효하고 저장 안 함.
   영구 저장(코인·해금)과 분리 → 클라우드 세이브·인벤토리 부담 없음. `createGame`/새 판마다 초기화.
2. **봇 동등 적용**: 방어전엔 아군 봇이 있다. 강화·증강 모두 **봇 영웅에도 적용**돼야 판이 안 기운다
   (효과는 `h.augments`/`h.itemPlus`를 읽으므로 isBot 무관 자동 적용, 단 "선택 로직"은 봇용 자동화 필요).
   밸런스 시뮬(defense-boss-sim.mjs)도 강화·증강 인지 봇으로 재작성해야 재측정 가능.
3. **레이스는 유지**: 성장만 무한히 세지면 "무한 승리"라 지루하다. 난이도 곡선도 함께 재튜닝해
   더 재밌는 레이스로 만든다(성장이 세진 만큼 웨이브 압박도 상향).
4. **밸런스는 시뮬로**: 모든 계수는 아래 "시작값"에서 출발해 봇 16판 시뮬로 도달 웨이브·보스2 구간
   도달률을 보며 수렴시킨다. 봇은 완벽 조작이라 실플레이어 대비 -1~2웨 기준.

# 시스템 A — 아이템 강화

## 개념
상점(우물/canShop)에서 보유 아이템을 골드로 강화 도전. 성공하면 그 아이템 능력치 배율↑.
**실패해도 아이템은 절대 파괴하지 않는다**(런 즉사 방지 — 재미가 아니라 좌절). 실패 = 골드만 소모.

## 상태 모델
- `h.itemPlus: number[]` — `h.items`와 인덱스 정렬된 강화 레벨 배열(각 0..N). 아이템은 id 문자열 유지
  (기존 "얕은 복사·스프레드 금지" 제약 보존). 판매/조합 시 해당 슬롯의 plus도 함께 이동/제거.
- 능력치 반영: `sumStats`가 각 아이템 기여분에 `(1 + plus × PLUS_STAT)`을 곱하도록 확장,
  또는 별도 `enhanceBonus(h)`를 더함. 강화 성공 시 `h.bonus = sumStats(...)` 재계산(buyItem과 동일 경로).
- ⚠️ 능력치 상한 주의: `CAPS`(def 0.6, atkSpeed 0.5, cdr 0.45)가 있어 def/공속/쿨감 아이템 강화는
  상한에 막혀 효율↓. → **강화는 무제한 스탯(atk/power/hp/regen/lifesteal)에 의미가 크다**를 UI로 안내.

## 성공 확률 · 비용 · 강화량 (시작값 — 시뮬 튜닝 대상)

| 강화 | 성공률 | 누적 성공확률 | 비용(골드) | 스탯 배율 |
|---|---|---|---|---|
| +1 | 95% | 95% | 300 | +7% |
| +2 | 85% | 81% | 450 | +14% |
| +3 | 75% | 61% | 600 | +21% |
| +4 | 62% | 38% | 800 | +28% |
| +5 | 50% | 19% | 1000 | +35% |
| +6 | 42% | 8% | 1300 | +42% |
| +7 | 35% | — | 1600 | +49% |
| +8~ | 30%(바닥 28%) | — | +400/레벨 | +7%/레벨 |

- `PLUS_STAT = 0.07`(레벨당 +7%). 하드캡 **+10**(안전장치), 실질 상한은 비용·확률 경제가 만든다.
- **실패 정책**: 골드만 소모, 아이템 유지. **소프트 천장(pity)**: 같은 아이템 연속 실패마다 다음 성공률
  +5%p(성공 시 리셋) → 극단적 불운 방지, 도박 스트레스 완화.
- **골드 배출 목적**: 5칸 다 찬 뒤 남는 골드(보스 처치 웨이브×25 팀 골드로 계속 쌓임)를 흡수.
  비용 곡선은 후반 골드 수입과 맞물려 "매 5웨마다 +1~2강" 정도 속도가 되게 조정.

## 봇 처리
`botShop`(engine.js:4561) 확장: 인벤토리 꽉 + 여유 골드(예: ≥1200) 있으면, 가장 강화 낮은
핵심 아이템(공격/주문 계열 우선)을 1회 강화 시도. 사람과 같은 확률·비용 적용.

## UI
상점 화면(RiftGame.jsx)에서 각 아이템 슬롯에 "강화(+N)" 버튼 + 비용·성공률 표시.
성공/실패 연출(반짝/깨짐 없는 부드러운 실패 피드백) + 사운드. 되돌리기 스택(shopStack)과는 무관
(강화는 확정 — undo 불가, 명시).

## 밸런스 노브
`PLUS_STAT`(레벨당 배율) ≫ 성공률 곡선 ≫ 비용 곡선. 스탯 배율이 가장 민감 — 0.07→0.10이면 후반 급강.

# 시스템 B — 조디악 증강 카드

## 개념
5의 배수 파도를 넘길 때마다(그림자 정예/보스가 나오는 그 파도) **증강 카드 3장 중 1장 선택**(롤 증강식 —
죽은 뽑기 없음). 카드는 능력치가 아니라 **규칙을 바꾸는 효과**로 런마다 빌드가 달라지게 한다(하데스 신뽑기).

## 상태 모델
- `h.augments: string[]` — 획득한 증강 id 목록. 효과는 훅 지점에서 `h.augments` 조회로 적용.
- 스택형(중복 획득 시 강해짐)과 유니크형(1회)을 카드별로 정의. 런 한정, 저장 안 함.

## 뽑기 시점 / 페이싱
- `stepDefenseWaves`(engine.js:2974)에서 `w % 5 === 0` 파도를 **클리어**한 직후 뽑기 트리거
  (그림자 리듬과 일치, 15·30웨 등 클라이맥스와 자연 연동).
- 뽑기 순간 시뮬 일시정지(또는 짧은 선택 창) + 카드 3장 모달. 사람만 선택 UI, 봇은 자동 선택.

## 희귀도 · 노출 빈도 (시작값)
| 희귀도 | 기본 가중치 | 성격 |
|---|---|---|
| 일반(Common) | 62% | 단순 스탯/조건 효과 — 안정적 성장 |
| 레어(Rare) | 30% | 조건부·빌드 조력 효과 |
| 전설(Legendary) | 8% | 빌드를 정의하는 런 체인저 |

- **픽3 보정**: 3장 중 **최소 1장은 레어+ 보장**(전부 일반 방지).
- **깊은 웨이브 보상**: 20웨 이후 10웨마다 전설 가중치 +1%p(장기 런에 대박 순간↑).
- **불운 방지(pity)**: 최근 6회 뽑기에 전설이 없으면 다음 뽑기에 전설 1장 보장.

## 카드 풀 구조 (출시 목표 ~40장)
- **일반 계열 풀**(~18): 공격/방어/유틸 3분류.
- **레어 계열 풀**(~12): 조건부·시너지.
- **전설 계열 풀**(~6): 공용 런 체인저.
- **조디악 시그니처**(12, 지신별 1장): 그 영웅의 `zodiacId`에게만 등장. 레어/전설급. **게임 정체성 축**
  (이름이 조디악 블리츠 — 12계열이 곧 리플레이 축, [`zodiac.js`](../src/shared/zodiac.js) 12지신).

### 예시 카드 (설계 방향 — 확정 아님)
**일반**: 평타 +12% · 최대 체력 +15% · 스킬 쿨 −8% · 이동 중 재생 · 처치 시 소량 골드
**레어**: 평타가 다음 적에게 튕김(연쇄) · 스킬 적중 시 궁 쿨 −1초 · 체력 30%↓일 때 받는 피해 −40% ·
골드 획득 +25%(→ 강화 시너지) · 웨이브 생존마다 영구 +공격력(이 런)
**전설**: 궁극기 쿨 −50% · 평타 2회 발동 · 스킬 사용 시 분신 1기 소환 · 받는 피해의 30%를 반사 ·
5초마다 강력한 자동 폭발
**조디안 시그니처(캡스톤)**: 용=스킬에 화염 도트 / 호랑이=3연타마다 강타 / 쥐=처치 시 분신 /
소=받는 피해 고정 감소 + 도발 / 토끼=회피 확률 / 뱀=중독 확산 … (12지신 각 1장)

## 엔진 훅 지점 (기존 시스템에 주입)
증강 효과는 기존 버프(dragonT/baronT)·`ARENA_CLASS_MOD` 배율과 같은 패턴으로 삽입:
- `damageHero`: 주는/받는 피해 배율, 온히트 효과, 피해 반사
- 처치 처리부(hero-death 핸들러): 처치 시 폭발/골드/스택
- `castAttack`/스킬 시전: 쿨 환급, 추가 투사체, 분신
- `stepHero`(틱): 재생·오라·스택 버프·자동 폭발
- `gainXp`/파도 생존: "웨이브당 영구 +스탯" 스택
- `castAttack` 평타: 연쇄/2회 발동

## 봇 처리
아군 봇 자동 선택: 역할 가중(공격봇→공격 계열 우선) 또는 희귀도 우선 랜덤. 엔진 효과는
`h.augments` 기반이라 봇에도 그대로 적용. 시뮬 봇도 자동 선택 로직 공유.

## UI
카드 3장 선택 모달(희귀도 색·이름·설명), 시뮬 일시정지. 획득 증강은 HUD 한켠에 아이콘 목록으로.

## 밸런스 노브
전설 카드 강도 ≫ 희귀도 가중치·pity ≫ 카드 개별 수치. 죽은 픽(안 뽑는 카드)·사기 조합을 시뮬+플레이로 색출.

# 두 시스템의 상호작용
- "골드 획득 +25%" 증강 → 강화 자원 가속. "웨이브당 +공격력" → 강화와 곱연산으로 후반 파워 급증 가능
  → **곱연산 폭주 감시**(레이스 유지 원칙). 필요 시 증강×강화 동시 상한 또는 난이도 상향으로 상쇄.
- 둘 다 런 한정·봇 적용·난이도 재튜닝이라는 공통 축을 공유 → 시뮬 하네스 한 번 손보면 둘 다 측정 가능.

# 구현 단계
### 1단계 — 아이템 강화 (작고 자기완결적, 즉효)
- `h.itemPlus` 상태 + `sumStats` 강화 반영 + 재계산 경로
- 강화 함수(성공률·비용·pity·실패=골드만) + 상점 UI 버튼/연출
- `botShop` 자동 강화 + 난이도 소폭 재튜닝 + 시뮬 재측정

### 2단계 — 조디악 증강 (대형)
- `h.augments` 상태 + 뽑기 트리거(5배수 파도 클리어) + 픽3 모달 + 봇 자동선택
- 카드 풀 데이터(희귀도·효과·시그니처 12) + 엔진 훅 주입 지점별 구현
- pity/가중치 + HUD 증강 목록 + 난이도 재튜닝 + 시뮬 재측정

# 열린 질문 / 리스크
- 강화 상한: 하드캡(+10) vs 순수 경제 소프트캡 — 시뮬로 후반 곡선 보고 결정.
- 증강 뽑기 중 일시정지 방식(완전 정지 vs 짧은 창) — 방어전 파도 압박과 충돌하지 않게.
- 곱연산 폭주(증강×강화) — 동시 상한/난이도 상향 중 택.
- 스코프: 우선 **방어전 전용**. 강화는 추후 콜로세움 등 확장 여지 있으나 이번은 방어전 벽 해소에 집중.
- 밸런스 시뮬: 봇에 강화·증강 자동선택을 넣어야 도달 웨이브 재측정 가능(현 하네스 확장 필요).
