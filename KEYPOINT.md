# BoardGameParty — 핵심 기획 (Keypoint)

> 보드/파티게임을 계속 추가해서 함께 즐기기 위한 게임 모음 프로젝트.
> 첫 번째 샘플 게임은 **사다리 게임**(주사위 굴려 골인하는 뱀사다리형 보드게임).

---

## 0. 프로젝트 전체 방향

- **성격**: 여러 보드/파티게임을 하나의 사이트에 모아두고 같이 즐기는 컬렉션.
- **확장성**: 게임을 계속 추가할 수 있는 구조여야 함 (게임 목록 → 게임 선택 → 플레이).
- **타깃 기기**: 7인치 휴대용 화면, **가로(landscape) 16:9 ~ 16:10 와이드** 기준 설계.
- **타깃 사용자**: 아이들 포함 가족/친구. → 직관적이고 너무 가혹하지 않은 난이도.

## 1. 기술/플레이 결정 사항 (확정됨)

| 항목 | 결정 |
|------|------|
| 기술 스택 | **React + Vite** |
| 플레이 방식 | **한 기기에서 핫시트(번갈아 플레이)** |
| 캐릭터 그래픽 | **이모지/유니코드로 시작** (나중에 이미지로 교체 가능하게) |
| 화면 방향 | 가로 와이드 고정 (세로일 때 안내) |

## 2. 첫 게임: 사다리 게임 — 핵심 규칙

- 시작 시 **플레이 인원 수 선택** → 각자 **12지신 말 선택**.
- 자기 턴에 **주사위를 굴림** → 나온 숫자만큼 앞으로 이동.
- **올라가는 발판 / 내려가는 발판**이 보드 곳곳에 존재.
  - 내려가는 발판은 **너무 많이 내려가지 않게** (아이들 좌절 방지).
- **먼저 골인(GOAL)하는 사람이 우승**.

### 말 (12지신)
- 총 **12개** 캐릭터: 쥐 🐭 / 소 🐮 / 호랑이 🐯 / 토끼 🐰 / 용 🐲 / 뱀 🐍 / 말 🐴 / 양 🐑 / 원숭이 🐵 / 닭 🐔 / 개 🐶 / 돼지 🐷.
- 시작 화면에서 인원 선택 후 각자 하나씩 고름 (중복 선택 불가).

### 주사위
- 굴릴 때 **주사위가 굴러가는 연출** 필요.
- 멈추면 **최종 숫자를 화면 중앙에 큰 애니메이션**으로 표시.

### 보드 모양
- grid만 쓰면 진행 방향이 헷갈림 → **명확한 뱀(S자, boustrophedon) 형태**로 경로 표시.
- 칸 번호와 진행 방향이 한눈에 보이도록.

### 발판(사다리/미끄럼틀)
- **올라가는 발판(↑)**: 위쪽 칸으로 점프.
- **내려가는 발판(↓)**: 아래쪽 칸으로 이동하되 **하강 폭 제한**.

---

## 3. 규칙 확정값 (결정됨)

| 항목 | 확정값 | 비고 |
|------|--------|------|
| 칸 수 | **30칸** (5×6 뱀형) | 추후 **칸 수 선택 가능**하게 (설정패널) |
| 승리 조건 | **초과 허용(overshoot)** | 골 칸 이상이면 골인 |
| 주사위 개수 | **1개 (1~6)** | |
| 최대 인원 | **5명** | 최소 2명 |
| 사운드 | **포함** | 주사위/이동/골인 효과음 + on/off 토글 |
| 보너스 룰(6→한번더) | **없음** | |
| 턴 순서 | 말 선택 순서대로 | |
| 발판 개수/폭 | 올라감 ~4개(+3~+8), 내려감 ~3개(−1~−3, 제한) | board.config로 데이터화 |

> **설정패널(향후)**: 위 1~5번(칸 수 / 승리 조건 / 주사위 / 인원 / 사운드)을
> 게임 시작 전 설정패널에서 조정 가능하게. **1차 MVP에는 미포함**,
> 단 모든 값을 `board.config.js` 등 데이터로 분리해 패널만 붙이면 되게 설계.

### 연출 / UX
- **이동 애니메이션**: 말이 칸을 **한 칸씩 따라 이동** (순간이동 X).
- **발판 발동 연출**: 올라감/내려감을 시각적으로 강조.
- **현재 턴 표시**: 누구 차례인지 명확히.
- **다시하기 / 처음으로**: 게임 종료 후 재시작 버튼.
- **세로 방향 안내**: 세로로 들면 "가로로 돌려주세요" 안내.

### 기타
- **언어**: 한국어 UI.
- **에셋 교체 용이성**: 이모지 → 추후 PNG/SVG 캐릭터로 교체 가능한 구조.
- **상태 저장**: 초기엔 새로고침 시 초기화(저장 없음). 추후 검토.
